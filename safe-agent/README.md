# Safe Automation Agent

A lightweight, safety-first personal automation system that lets you control your laptop from your phone through a cloud relay.

```
 ┌──────────┐       ┌─────────────────┐       ┌──────────────────┐
 │  Phone   │──────▶│  Cloud Agent    │◀──────│  Laptop Listener │
 │ Telegram │       │  (FastAPI)      │ poll  │  (Python script) │
 └──────────┘       └─────────────────┘       └──────────────────┘
     /run cmd    validates & queues      fetches & executes
                  whitelisted actions    whitelisted functions
```

## Safety Design

### Non-Negotiable Rules

| Rule | How it's enforced |
|------|-------------------|
| No arbitrary shell execution | Actions are predefined Python functions; no `shell=True` |
| Strict whitelist | Triple-checked: Telegram bot → Cloud agent → Laptop listener |
| No password/sensitive access | Actions only read system metrics or open URLs |
| No self-modifying code | Zero use of `eval()`, `exec()`, or dynamic imports |
| No autonomous browsing | No HTTP requests from actions; only the listener talks to cloud |
| Explicit user commands only | Every action requires a `/run` command from an authorized user |

### Authentication

- **Cloud ↔ Laptop**: Shared secret token in `X-Auth-Token` header on every request.
- **Phone ↔ Bot**: Telegram chat ID allow-list — only your account can send commands.

## Setup Guide

### Prerequisites

- Python 3.10+
- A Telegram account (for phone control)
- A Telegram bot token from [@BotFather](https://t.me/BotFather)

### Step 1: Create a Telegram Bot

Follow the full guide: **[docs/TELEGRAM_SETUP.md](docs/TELEGRAM_SETUP.md)**

Quick version:
1. Message @BotFather on Telegram → `/newbot` → copy the token.
2. Message your new bot `/start`, then use the API to find your chat ID.

### Step 2: Configure Environment

```bash
cd safe-agent
cp .env.example .env
# Edit .env with your values:
#   SAFE_AGENT_TOKEN   — auto-generated by start.sh, or run:
#                        python -c "import secrets; print(secrets.token_urlsafe(32))"
#   TELEGRAM_BOT_TOKEN — from @BotFather
#   ALLOWED_CHAT_IDS   — your Telegram chat ID (use /myid to find it)
```

The `.env` file is auto-loaded by `python-dotenv` — no manual `export` needed.

### Step 3: Deploy the Cloud Agent

#### Option A: GitHub Codespaces (recommended)

```bash
cd safe-agent
./start.sh
```

The launch script will:
- Check for `.env` (create from template if missing, auto-generate token)
- Install any missing Python packages
- Start the cloud agent + Telegram bot
- Clean up both on Ctrl+C

The Codespace will auto-forward port 8000. Copy the forwarded URL (e.g., `https://your-codespace-8000.app.github.dev`) — you'll need it for the laptop listener.

#### Option B: Google Colab

```python
# In a Colab notebook cell:
!pip install fastapi uvicorn python-telegram-bot httpx

import os
os.environ["SAFE_AGENT_TOKEN"] = "your-token-here"
os.environ["TELEGRAM_BOT_TOKEN"] = "your-bot-token"
os.environ["ALLOWED_CHAT_IDS"] = "your-chat-id"

# Upload safe-agent/ files or clone the repo
# Then run:
!python -m cloud.agent &
!python -m telegram_bot.bot
```

Use [ngrok](https://ngrok.com/) or Colab's port forwarding to expose port 8000.

#### Option C: Any Cloud Host (VPS, Railway, Render)

```bash
git clone <your-repo>
cd safe-agent
pip install -r requirements.txt
export $(grep -v '^#' .env | xargs)
python -m cloud.agent &
python -m telegram_bot.bot
```

### Step 4: Run the Laptop Listener

On your laptop:

```bash
cd safe-agent
pip install -r requirements-laptop.txt

# Set environment variables
export SAFE_AGENT_TOKEN="same-token-as-cloud"
export CLOUD_AGENT_URL="https://your-codespace-8000.app.github.dev"

# Start listening
python -m laptop.listener
```

The listener will poll every 3 seconds (configurable via `POLL_INTERVAL`).

### Step 5: Test from Your Phone

1. Open Telegram and message your bot.
2. Send `/start` to verify it's running.
3. Send `/help` to see available actions.
4. Send `/run ping` to test the full pipeline.
5. Send `/run get_system_status` to see laptop metrics.

## Telegram Commands

### Shortcut Commands (one-tap from phone)

| Telegram Command | Maps to Action | Description |
|-----------------|----------------|-------------|
| `/status` | `get_system_status` | Laptop CPU, RAM & disk |

| Command | Description |
|---------|-------------|
| `/start` | Welcome message |
| `/help` | Full command reference |
| `/myid` | Show your Telegram chat ID (for config) |
| `/health` | Check cloud agent status |
| `/run <action>` | Run any whitelisted action by name |

## Adding New Actions

1. **Define the function** in `laptop/actions.py`:
   ```python
   def my_new_action() -> str:
       """Description of what it does."""
       # Your safe, read-only logic here
       return "result string"
   ```

2. **Register it** in `laptop/actions.py` → `ACTION_REGISTRY`:
   ```python
   ACTION_REGISTRY["my_new_action"] = my_new_action
   ```

3. **Whitelist it** in `shared/config.py` → `WHITELISTED_ACTIONS`:
   ```python
   "my_new_action": {
       "description": "What it does.",
       "function": "my_new_action",
       "args": [],
   },
   ```

4. Restart the cloud agent and laptop listener.

## Resource Usage

| Component | RAM | CPU |
|-----------|-----|-----|
| Cloud Agent (FastAPI) | ~30 MB | Minimal (async) |
| Telegram Bot | ~25 MB | Minimal (long-polling) |
| Laptop Listener | ~10 MB | Near-zero (sleeps between polls) |

All three combined use under 100 MB — well within 4 GB laptop constraints.

## Limitations

- **Polling latency**: Commands take up to `POLL_INTERVAL` seconds to reach the laptop. This is a deliberate trade-off for simplicity and NAT traversal.
- **No file transfer**: Actions return text strings only (max 4 KB). For file transfer, extend with a dedicated upload endpoint.
- **Single laptop**: Designed for one laptop listener. To support multiple, add a device ID field.
- **In-memory queue**: If the cloud agent restarts, pending actions are lost. For persistence, swap the deque for SQLite.
- **No HTTPS built-in**: Rely on the hosting platform (Codespaces, Render, etc.) for TLS termination.
